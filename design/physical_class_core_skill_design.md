# 物理职业核心技设计稿

## 设计目标

- 本文档统一收敛当前物理职业的核心技设计，作为后续 `Feat -> Runtime Skill -> 5e Meta` 实现的规则源。
- 目标是在保留 D&D 5e 风味的前提下，对职业核心技做足够简化，使其适配当前自动战斗与轻量技能系统。
- 本文档只定义职业核心技，不展开完整 Lv1-Lv5 Build 树；后续各职业 Build 文档应以此为底盘继续细化。
- 所有能力优先复用各职业的基础攻击母技能，避免另起独立伤害链。

## 总体原则

- 5e 核心公式保持不变：命中仍然采用 `d20 + hit bonus vs AC`，伤害仍然基于现有基础攻击链结算。
- 职业核心技优先通过以下方式表达差异：
  - 触发时机
  - 目标筛选
  - 命中前修正
  - 命中后附伤
  - 反应登记与后结算
- 自动战斗场景下，职业技能设计必须和目标选择倾向同时定义；只设计结算规则而不设计目标选择，会导致职业手感显著失真。
- 所有成长入口统一由 `Feat` 承载，后续落地时只使用 `grant_skill`、`modify_skill`、`replace_skill`。
- 每个物理职业的完整战斗套件统一收敛为 `4` 个技能位：
  - `普攻`
  - `核心被动`
  - `主动`
  - `大招`
- 其中 `Lv1` 固定获得 `普攻 + 核心被动`，用于建立开局职业辨识度；`主动` 与 `大招` 在后续成长节点解锁。
- 前四个职业的核心身份主要落在 `核心被动` 上；`圣骑` 的签名体验主要落在 `主动` 上，`野蛮人` 则通过 `核心被动 -> 大招` 的联动完成终局爆发。
- `额外攻击`、`额外行动`、`命中后追击` 必须严格区分，避免实现时语义串线。

## 母技能映射

- 战士：近战基础武器攻击
- 武僧：徒手打击
- 盗贼：轻巧近战攻击
- 游侠：远程基础武器攻击
- 圣骑：近战基础武器攻击
- 野蛮人：近战基础武器攻击

## 四技能结构

- 所有物理职业都采用统一的 `4 技能位` 结构：
  - `普攻`：基础攻击母技能，`runtime.active`
  - `核心被动`：定义职业主循环，`runtime.passive`
  - `主动`：主力手动/自动施放技能，`runtime.active`
  - `大招`：终局高潮技能，通常为 `runtime.active`，也可在极少数职业上定义为强识别度 `runtime.passive`
- 推荐解锁顺序：
  - `Lv1`：普攻 + 核心被动
  - `Lv3`：主动
  - `Lv5` 或终局节点：大招
- 设计目的：
  - 开局立即建立职业辨识度
  - 中期提供明确的爆发按钮
  - 后期补足高潮时刻和终结体验
  - 保持自动战斗下的技能结构可理解、可预测

## 职业核心技

### 战士：反击

#### 职业核心技

- `反击` 的职业定位是站桩反打与近战压制。
- 该能力不依赖敌方是否命中，只要敌方主动对战士发动近战武器攻击，就会触发回手。

#### 触发规则

- 触发条件：敌方对你发动一次近战武器攻击。
- 命中要求：无论该次攻击命中或未命中，均可触发。
- 结算时机：敌方本次攻击动作完全结算后，战士立即执行一次反击。
- 次数限制：不限制次数。
- 防递归规则：
  - 反击本身不能再次触发反击。
  - DOT、反伤、溅射、连锁伤害不触发反击。
  - 一次攻击事件只允许触发一次反击。

#### 目标选择倾向

- 主动出手时优先选择近战敌人。
- 次优先选择攻击频率高、多段攻击或高威胁前排敌人。
- 若多个目标权重相近，优先选择当前最可能在敌方回合继续攻击自己的目标。

#### 数值口径

- 伤害来源：完全复用近战基础武器攻击链。
- 反击默认不附带额外伤害系数，先以频率取胜。
- 若后续需要压平强度，优先下调反击伤害系数，不改触发频率。

#### 实现备注

- 运行时建议使用“先登记、后结算”的反应技语义。
- 触发点应挂在“成为近战武器攻击目标”事件上，而不是挂在“受伤”事件上。
- 需要单独的反击来源标记，避免双战士或多反击单位之间出现递归连锁。
- Web 表现应先播放敌方动作，再播放战士回手，日志也应保持同样顺序。

### 武僧：连击

#### 职业核心技

- `连击` 的职业定位是高频追打与持续压制。
- `连击` 的本质是概率触发的额外攻击，不是轻击附伤，也不是新的完整攻击行动。

#### 触发规则

- 触发条件：武僧的徒手打击命中目标后。
- 触发概率：`50%`
- 结算时机：当前命中结算完成后，以 `50%` 概率对同一目标追加 `1` 次额外攻击。
- 次数限制：不限制次数。
- 防递归规则：
  - 连击生成的额外攻击不能再次触发连击。
  - 连击不是额外行动，不允许重新选择目标。

#### 目标选择倾向

- 优先攻击低 AC、低血量、容易命中的目标。
- 次优先攻击已经受伤、适合连段收割的目标。
- 不优先攻击高 AC 的硬目标，避免因为持续落空而损失连击收益。

#### 数值口径

- 连击追加的是一次标准额外攻击，应复用正常徒手攻击伤害结算。
- 当前推荐概率固定为 `50%`。
- 连击的强度来源是“命中后有概率再打一拳”，而不是额外附伤倍率。

#### 实现备注

- `连击` 应建模为命中后触发的概率额外攻击。
- 这次额外攻击属于同一次攻击流程中的追加攻击，不是新的完整行动。
- 为避免无限套娃，连击追加出的额外攻击不得再次触发连击。
- 自动战斗 AI 需要显式提高“易命中目标”权重，否则武僧会因为错误选目标而丢失职业手感。

### 盗贼：伏击

#### 职业核心技

- `伏击` 的职业定位是抓取牵制窗口，对被分神或被围攻的目标打出额外伤害。
- 本设计不引入完整背刺朝向、潜行或 Advantage 系统，统一通过战场牵制条件来表达。

#### 触发规则

- 满足以下任一条件时触发：
  - 目标当前攻击目标不是你。
  - 目标本回合已被其他友军攻击过。
- 结算方式：你对该目标造成额外伤害。
- 次数限制：不限制次数。
- 边界规则：
  - “其他友军攻击过”必须排除盗贼自己。
  - 伏击是附加伤害，不是额外一击。
  - 若本次为双武器攻击，整次双持动作只结算 `1` 次伏击伤害，不会因为主副手分别命中而触发两次。

#### 目标选择倾向

- 优先攻击满足伏击条件的目标。
- 在满足条件的目标中，优先选择血量更低、护甲更低、可快速击杀的目标。
- 若当前场上没有满足条件的目标，再退回常规补刀逻辑。

#### 数值口径

- 因为不限制次数，伏击附伤不应采用过高爆发模板。
- 推荐使用稳定额外伤害，而非极高波动骰组。
- 伏击的强度来源应主要来自“条件常驻后持续增伤”，而不是单次斩杀倍率。

#### 实现备注

- 自动战斗目标评分必须直接读取伏击条件，否则盗贼会频繁选错目标。
- 需要在回合运行时记录“目标本回合是否被其他友军攻击过”。
- 条件判定应尽量简单透明，避免引入站位朝向和背刺角度系统。
- UI 或日志建议明确显示伏击触发原因，如“目标被牵制”或“目标分神”。

### 游侠：标记

#### 职业核心技

- `标记` 的职业定位是锁定单个猎物，并通过持续追击获得稳定增伤。
- 标记同时只能存在一个，强调游侠的单目标猎杀身份。

#### 触发规则

- 触发条件：游侠的攻击命中目标后。
- 施加效果：给目标附加 `标记`。
- 持有上限：同一名游侠同时只能维持 `1` 个标记。
- 覆盖规则：若你已标记其他目标，则新标记覆盖旧标记。
- 追加收益：你再次攻击被你标记的目标时，造成额外伤害。
- 基础版收益只对游侠本人生效，不默认外溢给队友。

#### 目标选择倾向

- 若已有标记目标且目标仍存活，优先继续攻击该目标。
- 若尚无标记，则优先选择高威胁、较耐打、不会被瞬间清掉的目标。
- 不优先把标记挂在即将死亡的小怪身上。

#### 数值口径

- 第一次命中负责上标记，不额外叠多层收益。
- 后续对标记目标的增伤建议采用固定伤害或小额稳定附伤。
- 标记的核心收益来自“持续追击同一目标”，而不是“全场撒网”。

#### 实现备注

- 标记归属必须绑定游侠自身，不可做成公共 debuff。
- 多个游侠同时存在时，应各自维护自己的当前标记目标。
- 自动战斗 AI 需要显式提高“当前标记目标”的选择权重，否则单标记机制会被自身 AI 抵消。
- 若后续需要扩展团队协同，应通过强化 feat 单独增加“队友攻击标记目标时获得小额收益”，不放入基础版核心技。
- `狩猎指引` 的收益应动态跟随你当前的标记目标；若当前没有标记目标，AI 不应施放该技能。

### 圣骑：破邪斩

#### 职业核心技

- `破邪斩` 的职业定位是稳定的近战神圣裁决，并兼具拆除敌方强化的能力。
- 在四技能结构中，`破邪斩` 被定义为圣骑的签名主动技能，而不是常驻被动。

#### 触发规则

- 触发方式：主动施放，推荐 `CD3`。
- 施放效果：对当前目标发动一次近战武器攻击。
- 结算顺序：
  - 先结算正常武器伤害。
  - 再追加神圣伤害。
  - 最后驱散目标 `1` 个正面增益。
- 强化空间：对 `undead`、`fiend` 等邪恶单位可额外追加神圣特攻伤害。
- 次数限制：由技能冷却控制，不做额外次数限制。

#### 目标选择倾向

- 优先攻击带有正面 buff 的敌人。
- 次优先攻击 `undead`、`fiend` 等可触发额外收益的目标。
- 若没有以上目标，则优先处理高威胁前排或关键单体。

#### 数值口径

- 神圣伤害建议使用稳定附伤，不依赖额外资源。
- 驱散数量固定为 `1`，不建议扩大为全驱散。
- 邪恶特攻可通过额外固定伤害或单独附伤段表达。

#### 实现备注

- `破邪斩` 应作为独立主动 skill 出现在技能列表中。
- 驱散建议放在命中后结算，避免和 AC、减伤、命中修正类 buff 产生前置时序争议。
- 需要统一正面 buff 的筛选规则，避免误删中立或不可驱散状态。
- 自动战斗 AI 需要额外识别“带 buff 目标”，否则破邪斩的拆解价值无法稳定体现。

### 野蛮人：重击

#### 职业核心技

- `重击` 的职业定位是牺牲部分精准，换取更高的基础伤害和更高的暴击频率。
- 在四技能结构中，`重击` 被定义为野蛮人的单次爆发主动，而终局 `狂暴` 由 `狂怒` 层数驱动自动触发。

#### 触发规则

- 触发方式：主动施放，推荐 `CD2`。
- 施放效果：对当前目标发动一次强化近战攻击。
- 本次攻击生效内容：
  - 降低命中修正
  - 提高基础伤害
  - 扩大暴击范围
- 次数限制：由技能冷却控制，不做额外次数限制。

#### 目标选择倾向

- 优先攻击低 AC、高血量、适合持续重压的目标。
- 不优先攻击极高 AC 的敌人，避免命中惩罚导致持续落空。
- 若存在已残血目标且命中率足够，也可切换为收割。

#### 数值口径

- 推荐基础版：`命中 -2 / 伤害 +3 / 暴击范围 19-20`
- 若后续需要强化，可逐步提高伤害或继续扩大暴击范围，但不建议一开始直接进入 `18-20`。
- 暴击高光应作为职业特例处理，不应顺势放开为全局规则。

#### 实现备注

- `重击` 应作为独立主动 skill 出现在技能列表中。
- `重击` 的核心是命中修正、基础伤害修正和暴击阈值修正三者同时生效。
- 自动战斗 AI 必须感知“难命中目标”并规避高 AC 敌人，否则职业体验会严重打折。
- 若项目当前仍坚持“只有 nat20 才暴击”的全局约束，则需明确为野蛮人职业特例。
- `狂暴` 触发后应锁定本次状态窗口：持续期间不再积累狂怒，且每场战斗最多触发 `1` 次。

## 自动战斗目标选择总则

- 自动战斗目标选择应采用简单打分模型，而不是复杂前瞻搜索。
- 推荐基础打分项：
  - `base_threat`
  - `finish_bonus`
  - `class_synergy_bonus`
  - `marked_target_bonus`
  - `buff_dispel_bonus`
  - `easy_hit_bonus`
- 每个职业主要通过 `class_synergy_bonus` 调整自身倾向，不改公共框架。
- 若目标选择逻辑与职业核心技语义冲突，应优先修正 AI 权重，而不是先砍技能设计。

## 实现落点建议

- `design/`：以本文档为规则源，回写各职业 Build 文档。
- `config/feat_build_config.lua`：定义职业核心技对应的 Feat。
- `config/class_build_progression.lua`：决定职业何时拿到核心技与后续强化。
- `config/skill_runtime_config.lua`：定义被动、命中后附伤、反应登记、标记等运行时语义。
- `config/skill_5e_meta.lua`：补齐 5e 口径与规则说明。
- `skills/`：实现各职业核心技的具体技能逻辑。
- `modules/`：只补必要的目标事件、回合标记和自动战斗目标评分支持。

## 风险与边界

- 战士 `反击`、武僧 `连击`、盗贼 `伏击` 都不限制次数，后续平衡应优先控制单次收益，不要反向削成无触发感。
- 游侠 `标记` 若 AI 不会持续追标，职业会失去核心手感。
- 圣骑 `破邪斩` 若驱散过强，会挤压控制与辅助职业空间，因此必须固定为驱散 `1` 个正面 buff。
- 野蛮人 `重击` 若不配套自动战斗选目标，实战命中会过低，导致纸面设计和实际体验严重脱节。
