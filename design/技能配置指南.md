# MiniBattleSimulator 技能配置与开发指南

本文档专门针对 `MiniBattleSimulator` 中的**技能系统**提供详尽的配置和开发指南。
技能系统采用了**“静态数据（JSON）+ 动态行为（Lua）”**相分离的双层架构，以实现高度的可扩展性和配置灵活性。

---

## 一、 静态基础配置 (`res_skill.json`)

静态数据统一配置在 `config/res_skill.json` 中。它主要由数值策划负责，定义了技能的面板属性、释放条件和数值参数。

### 1.1 核心字段解析
| 字段名 | 说明 | 示例 |
| :--- | :--- | :--- |
| **`ID`** | 技能的唯一标识。通常为 9 位数字（`ClassID` + 占位符 + `Level`）。 | `10000101` |
| **`ClassID`** | 技能的原型/派系 ID。同种技能的不同等级共享此 ID。 | `100001` |
| **`Type`** | 技能类型枚举。决定了系统对冷却和消耗的判定方式。 | `1`: 普攻（无CD无消耗）<br>`2`: 主动技（受CD限制）<br>`3`: 大招（受能量限制）<br>`4`: 被动技 |
| **`CoolDownR`** | 冷却回合数。 | `2` (2回合CD) |
| **`Cost`** | 释放技能消耗的能量值（通常用于大招）。 | `100` |
| **`SkillParam`** | 核心数值参数数组（万分比，`10000 = 100%`）。通常 `[0]` 是基础伤害倍率。 | `[15000, 0, 0]` (150%伤害) |
| **`Buff1` ~ `Buff5`** | 附带 Buff 配置数组。<br>格式: `[触发概率, 目标类型, 持续回合, BuffID, 叠加层数]`。 | `[5000, 2, 2, 9001, 1]` |

---

## 二、 动态行为配置 (`skill_{ID}.lua`)

动态配置位于 `config/skill/` 目录下（如 `skill_100001.lua`）。它由战斗策划或特效美术负责，定义了技能在时间轴上的具体表现与结算时机。

### 2.1 攻击范围与目标选择 (`targetsSelections`)
在 MiniBattleSimulator 中，“攻击范围”并不是一个距离数值，而是通过**“阵营 + 站位 + 过滤”**的规则隐式定义的。

```lua
targetsSelections = {
    castTarget = 2, -- 【基础范围】阵营选择
    tSConditions = {
        Num = 3,    -- 【数量范围】作用目标数
        wpType = 1  -- 【位置范围】特定的站位或排数限制 (1,2,3 为前排)
    },
    tSFilter = {
        type = "HP_LOWEST" -- 【属性范围】特定的属性过滤 (选血最少的)
    }
}
```
**`castTarget` 枚举**：
*   `0`: 自身 (Self)
*   `1`: 友方 (Alias)
*   `2`: 敌方 (Enemy)
*   `3`: 除自己外的友方 (AlliesExcludeSelf)
*   `4`: 全场所有人 (EveryOne)

### 2.2 时间轴驱动模式 (`actData`)
这是最常用的技能配置方式，通过关键帧（KeyFrame）将逻辑结算与动画表现对齐。

```lua
actData = {
    {
        atLeastTimeS = 2.0, -- 该阶段最小持续时间（秒）
        keyFrameDatas = {
            {
                TriggerS = 0.5, -- 动作开始后 0.5 秒触发
                datatype = "DWCommon.LaunchBuff", 
                -- 注意：data 必须是 Lua 表的序列化字符串
                data = "{ buffId = 9002, targetType = 0 }" 
            },
            {
                TriggerS = 1.2,
                datatype = "DWCommon.DamageData",
                data = "{ damageRate = 12000, targetType = 2 }"
            },
            {
                TriggerS = 1.5,
                datatype = "DWCommon.LuaData",
                data = "{ LuaName = 'War/ActiveSkills/13194', FunctionName = 'a_skill_13194_edgeShowOn' }"
            }
        }
    }
}
```
**常用 `datatype` 事件与参数**：
*   **`DWCommon.DamageData` (触发伤害)**: 
    *   `damageRate`: 伤害倍率（万分比）。优先读取此值，否则读取 JSON 的 `SkillParam`。
    *   `cSVSkillAssociate`: 关联技能系数 ID（对应配置表第几项）。
*   **`DWCommon.LaunchBuff` (施加状态)**: 
    *   `buffId` / `AssociateBuff`: 触发时给目标挂载的 Buff ID。
    *   `targetType`: 目标类型（0:自身, 1:友方, 2:敌方等）。
*   **`DWCommon.LaunchSpell` (释放子技能/法术)**: 
    *   常用于多段攻击或发射飞行道具。由 `SkillExecutor.ExecuteSpell` 解析。
    *   包含 `SpellID`（子技能ID）、`TotalS`（持续时间）、`targetsSelections`（子技能独立寻敌逻辑）等参数。
*   **`DWCommon.LuaData` (脚本关键帧)**: 
    *   允许在时间轴任意点插入自定义 Lua 代码，极其灵活。
    *   `LuaName`: 脚本路径；`FunctionName`: 要调用的具体函数名。
    *   调用的函数会接收一个 `src` 上下文参数（包含施法者 `OwnerUnitID` 和原始配置 `OwnerSkillCsv`）。
*   **`DWCommon.SoundData` / `EffectData` (音效与特效)**: 
    *   `RelativePointName`: 特效挂载的骨骼点名称（如 `"Cha_RP_center"`）。

## 三、 高级脚本干预 (Programmer Mode)

对于无法用时间轴简单表达的复杂机制（如吸血、斩杀、根据战场环境变化），可以直接在脚本中定义生命周期回调函数，核心层会优先执行这些函数。

### 3.1 自定义释放条件 (`CheckCondition`)
在技能释放前检查特殊条件，如果不满足则无法释放。系统会在默认的 CD、能量、沉默状态检查之外，额外调用此函数。
```lua
function skill_100001.CheckCondition(hero, skill, condition)
    -- 示例1：只有在第3回合之后才能释放
    if BattleMain.GetCurrentRound() < 3 then return false end
    
    -- 示例2：检查目标身上的特定 Buff 层数是否符合要求
    local stackNum = BattleScriptExp.GetBuffStackNumBySubType(hero, 10002)
    return stackNum >= 3
end
```

### 3.2 自定义寻敌逻辑 (`SelectTarget`)
覆盖 `targetsSelections` 的默认逻辑，用于实现非常规的锁定目标机制。
```lua
function skill_100001.SelectTarget(hero, skill)
    -- 示例：使用高级 API 寻找攻击力最高的友方
    return BattleScriptExp.GetFriendSortByAttrId(hero, 9, false)
end
```

### 3.3 完全接管执行 (`Execute`)
只要声明了此函数，底层引擎就会**跳过 `actData` 解析**，直接运行你的代码。
**注意**：使用此模式时，你需要自己处理技能的冷却（`SetSkillCurCoolDown`）和能量消耗，因为系统会认为你已经完全接管了该技能的所有流程。

```lua
function skill_100001.Execute(hero, targets, skill)
    -- 1. 自定义分段伤害逻辑
    for _, target in ipairs(targets) do
        -- 第一段：真实伤害（例如扣除当前生命值 10%）
        local currentHp = target:GetAttr(E_ATTR.HP)
        local damage1 = math.floor(currentHp * 0.1)
        BattleScriptExp.MakeDmgReal(hero, target, damage1)
        
        -- 第二段：常规公式伤害
        BattleFormula.CalcDamage(hero, target, 15000, skill)
    end
    
    -- 2. 注入一个临时被动：下次受击时爆炸
    BattleScriptExp.InsertPassiveSkill(hero, targets[1], 1001, "OnReceiveDamage_Explode", E_TRIGGER_TIMING.ON_RECEIVE_DAMAGE, true)
    
    -- 3. 跨回合自定义变量存储（记录技能释放次数）
    local castCount = BattleScriptExp.GetCustomValue(hero, "Skill_100001_CastCount") or 0
    BattleScriptExp.SetCustomValue(hero, "Skill_100001_CastCount", castCount + 1)
    
    return true -- 必须返回 true 表示执行成功
end
```

### 3.4 性能与开发避坑建议
*   **字符串解析开销**：`keyFrame.data` 是以字符串形式存储的 Lua 表，底层使用 `load("return " .. str)()` 动态解析。应尽量避免在同一帧内触发过多的 `LuaData` 关键帧，以防止 CPU 峰值。
*   **全局变量污染**：导出的技能脚本通常会定义一个全局变量名（如 `skill_131940201`）。如果脚本内没有显式 `return` 该表，系统会尝试从 `_G` 中按 ID 规则检索，需注意命名规范以防冲突。

## 四、 完整配置模板示例

为了便于理解，以下提供一个名为 **“天核裂变”** 的复杂多段技能完整配置模板。
**技能设定**：
*   静态属性：消耗 100 能量的大招，无冷却。基础伤害 150%。
*   第一段动作 (0.5秒)：给自己附加一个攻击提升的 Buff。
*   第二段动作 (1.2秒)：向敌方前排发射一枚法术子弹（通过 LaunchSpell 实现）。
*   自定义脚本接管：如果目标血量低于 30%，则直接造成真实伤害（斩杀机制）。

### 4.1 JSON 静态模板 (`res_skill.json` 截取)
```json
{
    "ID": 10000101,
    "ClassID": 100001,
    "Name": "天核裂变",
    "Type": 3,               // 3: 大招(终极技能)
    "Cost": 100,             // 释放消耗 100 能量
    "CoolDownR": 0,          // 大招受能量限制，通常无冷却
    "SkillParam": [15000, 0, 0], // 参数[0] = 15000 (150% 基础伤害倍率)
    
    // 附带Buff：100% 概率给自身(0)附加 3 回合的 9002 号攻击提升 Buff
    "Buff1": [10000, 0, 3, 9002, 1],
    
    "UnlockLevel": 1,
    "UnlockStar": 1
}
```

### 4.2 Lua 动态脚本 (`config/skill/skill_10000101.lua`)
```lua
-- 技能 10000101: 天核裂变 动态逻辑
local skill_10000101 = {
    
    -- 1. 目标选择规则：敌方前排 (优先攻击)
    targetsSelections = {
        castTarget = 2, -- 2: 敌方
        tSConditions = {
            Num = 3,    -- 最多 3 个目标
            wpType = 1  -- 1: 站位限制为前排
        }
    },

    -- 2. 动作时间轴配置
    actData = {
        {
            atLeastTimeS = 2.0, -- 该阶段动画总长度至少 2.0 秒
            keyFrameDatas = {
                -- 关键帧 1：0.5秒时，给自己加 Buff (读取 JSON 中的 Buff1 配置)
                {
                    TriggerS = 0.5,
                    datatype = "DWCommon.LaunchBuff",
                    data = "{ buffId = 9002, targetType = 0 }" 
                },
                -- 关键帧 2：0.5秒时，播放音效
                {
                    TriggerS = 0.5,
                    datatype = "DWCommon.SoundData",
                    data = "{ soundName = 'spell_cast.wav' }"
                },
                -- 关键帧 3：1.2秒时，发射一枚法术子弹 (调用子技能)
                {
                    TriggerS = 1.2,
                    datatype = "DWCommon.LaunchSpell",
                    data = "{ SpellID = 10000102, TotalS = 0.5 }"
                }
            }
        }
    }
}

-- 3. 自定义释放条件 (可选)
-- 必须在第 2 回合之后才能释放
function skill_10000101.CheckCondition(hero, skill, condition)
    return BattleMain.GetCurrentRound() >= 2
end

-- 4. 自定义执行逻辑 (覆盖默认 actData 中的伤害计算)
-- 注意：如果保留此函数，底层的 actData(除了伤害外) 可能需要你手动处理，或者你只在需要斩杀的子技能(Spell)里写这个逻辑。
-- 这里展示如何手动计算斩杀伤害。
function skill_10000101.Execute(hero, targets, skill)
    for _, target in ipairs(targets) do
        local hpPercent = target:GetAttr(E_ATTR.HP) / target:GetAttr(E_ATTR.MAX_HP)
        
        if hpPercent < 0.3 then
            -- 斩杀：目标血量低于 30%，造成相当于其当前生命值的真实伤害
            local realDmg = target:GetAttr(E_ATTR.HP)
            BattleScriptExp.MakeDmgReal(hero, target, realDmg)
            Logger.Log(hero.name .. " 触发了天核裂变的斩杀！")
        else
            -- 常规：调用底层公式，使用 150% 的倍率 (15000) 计算普通伤害
            BattleFormula.CalcDamage(hero, target, 15000, skill)
        end
    end
    
    -- 必须手动调用扣除能量和设置冷却，因为 Execute 完全接管了技能流程
    hero:CostEnergy(skill:GetCost())
    skill:SetSkillCurCoolDown()
    
    return true
end

return skill_10000101
```
