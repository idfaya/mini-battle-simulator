# 术士职业设计

## 设计目标

- 本文档记录 `术士（火）` 的职业设计，并以 `design/newbattle/caster_class_core_skill_design.md` 为规则源。
- 术士成长统一由 `Feat` 承载，不引入完整法术位、血脉资源条或复杂 DOT 叠层系统。
- 术士主轴是 `火焰弹 -> 余烬点燃 -> 灰烬爆燃 -> 烈焰风暴`，重点表达点燃铺垫和条件爆发。
- 本文档使用三阶子职业结构：`low / mid / high`。

## 字段口径

- `promotion_stage = low / mid / high`。
- `basic_attack_slot`：火焰弹。
- `core_slot`：余烬点燃。
- `mid_slot`：灰烬爆燃。
- `high_slot`：烈焰风暴。
- 5e 画像：`primary_ability = int`，`spell_ability = int`，`armor_formula = robe_dex`，`save_proficiency = will`（sorcerer 位）。

## 三阶规格

| 阶段 | 能力包 | 规则定义 |
| --- | --- | --- |
| `low` | 火焰弹 + 余烬点燃 | 对单体敌人施加一次标准火焰法术豁免对抗；火焰法术使目标未通过豁免后，为其附加燃烧伤害；目标已处于燃烧状态时不重复施加，仅刷新持续时间。 |
| `mid` | 灰烬爆燃 | `CD3`，对当前目标施放一次强化火焰法术；若目标已被点燃，则额外造成火焰伤害。 |
| `high` | 烈焰风暴 | `主动技能`，`CD5`；对全体敌人造成基础火焰伤害；命中时若目标已被点燃，则额外造成 `1d8` 火焰伤害并延长燃烧 `1` 回合；命中时若目标尚未被点燃，则按 `余烬点燃` 规则为其附加燃烧。本技能不消耗 `余烬点燃` 在当回合的额度。 |

## 核心循环

- `火焰弹` 负责稳定单体法术出手。
- `余烬点燃` 负责建立燃烧状态。
- `灰烬爆燃` 负责对已点燃目标进行单点爆发兑现。
- `烈焰风暴` 负责在多目标战斗中进行点燃扩散与爆燃兑现。

## 触发规则

- `余烬点燃` 在火焰法术使目标未通过豁免后触发。
- 目标已处于燃烧状态时，不重复施加新的燃烧实例，只刷新持续时间。
- `烈焰风暴` 可为未点燃目标附加燃烧，且不消耗 `余烬点燃` 当回合额度。
- `烈焰风暴` 对已点燃目标额外造成 `1d8` 火焰伤害并延长燃烧 `1` 回合。

## 目标选择倾向

- 普攻优先攻击尚未点燃的高威胁目标。
- `灰烬爆燃` 优先攻击已被点燃的目标。
- `烈焰风暴` 优先在敌方数量较多，或场上已有 `2` 名及以上点燃目标时释放。
- 单体残局中，优先维持 `点燃 -> 爆燃` 循环。

## 数值口径

- `余烬点燃` 建议起步为 `1d4` 燃烧伤害，避免 DOT 本身压过主动技能。
- `灰烬爆燃` 的额外收益建议固定为已点燃目标额外 `1d8`。
- `烈焰风暴` 的核心收益来自点燃联动，不做无条件全屏高伤。

## 实现备注

- 燃烧状态应带来源标记，避免多术士互相覆盖时语义混乱。
- `余烬点燃` 建议作为标准 DOT 状态，其他术士技能只检查目标是否已点燃。
- 自动战斗 AI 必须优先识别已点燃目标，否则 `灰烬爆燃` 会频繁打空联动价值。

## 风险点

- `点燃` 本体伤害不能过高，否则会压过 `灰烬爆燃` 与 `烈焰风暴`。
- `烈焰风暴` 不能允许重复叠加多个燃烧实例，否则会形成滚雪球。
- 术士不做控制主轴，避免和法师抢控场定位。
