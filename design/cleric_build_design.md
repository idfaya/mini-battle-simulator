# 牧师职业设计

## 设计目标

- 本文档记录 `牧师（神术）` 的职业设计，并以 `design/newbattle/caster_class_core_skill_design.md` 为规则源。
- 牧师成长统一由 `Feat` 承载，不引入完整法术位、长期光环、专注中断或复杂治疗列表。
- 牧师主轴是 `神圣火花 -> 神恩庇护 -> 治愈之言 -> 圣域祷言`，重点表达减伤托底、关键抬血和团队稳场。
- 本文档使用三阶子职业结构：`low / mid / high`。

## 字段口径

- `promotion_stage = low / mid / high`。
- `basic_attack_slot`：神圣火花。
- `core_slot`：神恩庇护。
- `mid_slot`：治愈之言。
- `high_slot`：圣域祷言。
- 5e 画像：`primary_ability = str`，`spell_ability = wis`，`armor_formula = medium_capped`，`save_proficiency = will`。

## 三阶规格

| 阶段 | 能力包 | 规则定义 |
| --- | --- | --- |
| `low` | 神圣火花 + 神恩庇护 | 对敌方目标施加一次标准远程神术豁免对抗；若目标为友军，则改为回复 `1d8` 生命。每回合 `1` 次，全队共享一次：本回合第一次有友军被攻击命中时，该次伤害减少 `1d6`。 |
| `mid` | 治愈之言 | `CD3`，为生命值最低的友军回复生命，并优先解除其 `1` 个负面状态。 |
| `high` | 圣域祷言 | `主动技能`，`CD3`，持续 `2` 回合；期间我方全体 `AC +1`，且每个友军每回合第一次受到的伤害减少 `1d6`；与 `神恩庇护` 同回合触发时，仅取较高的减伤值，不叠加。 |

## 核心循环

- `神圣火花` 负责团队血线稳定时的基础神术出手，并在友军受伤时转为应急单体治疗。
- `神恩庇护` 负责每回合第一次团队减伤托底。
- `治愈之言` 负责救急抬血与负面清除。
- `圣域祷言` 负责高压回合的团队防线。

## 触发规则

- `神恩庇护` 在每回合第一次有友军被攻击命中时触发。
- `神恩庇护` 每回合 `1` 次，且全队共享，不按每个友军各享一次实现。
- `治愈之言` 自动选择生命值最低的友军，若多个目标接近，优先带关键负面状态者。
- `圣域祷言` 持续 `2` 回合；其减伤与 `神恩庇护` 同回合触发时只取较高值，不叠加。

## 目标选择倾向

- 普攻在团队血线稳定时，优先攻击高威胁敌人或不死/邪恶单位。
- `治愈之言` 优先施放给生命值最低或身上有关键负面状态的友军。
- `圣域祷言` 优先在己方多人半血、前排承压或敌方准备爆发时释放。
- 若团队血量稳定且无紧急救场需求，牧师应优先保留 `治愈之言` 给下一次危险波峰。

## 数值口径

- `神恩庇护` 固定为 `1d6` 减伤。
- `治愈之言` 应是单体高质量治疗，并附带 `1` 个负面清除。
- `圣域祷言` 只做短时团队庇护，不做成长时常驻光环。

## 实现备注

- 牧师是唯一以友方目标决策为主的法系职业。
- `治愈之言` 建议内置生命最低友军选择，降低自动战斗目标判断负担。
- `圣域祷言` 建议建模为团队短时 buff，而不是场地区域效果。

## 风险点

- 牧师不能同时堆过高奶量、清 debuff、群体减伤，否则会成为绝对最优后排。
- `神恩庇护` 和 `圣域祷言` 的减伤不能叠加。
- 牧师不应替代法师控场、术士爆发或邪术师连锁输出。
