# 法系职业核心技设计稿

## 设计目标

- 本文档统一收敛当前法系职业的核心技设计，作为后续 `Feat -> Runtime Skill -> 5e Meta` 实现的规则源。
- 目标是在保留 D&D 5e 风味的前提下，对法系职业做足够简化，使其适配当前自动战斗与轻量技能系统。
- 本文档当前覆盖仓库内已有的四个法系职业：
  - 法师（冰）
  - 术士（火）
  - 牧师（神术）
  - 邪术师（雷）
- 所有能力优先复用统一母技能，避免引入完整法术位、专注槽、法术准备列表与复杂区域状态机。

## 总体原则

- 5e 核心公式保持不变：
  - 攻击型法术继续使用现有法术命中链。
  - 豁免型法术继续使用现有法术豁免链。
  - 伤害、减速、点燃、减伤等效果统一挂在既有战斗结算系统上。
- 法系职业差异优先通过以下方式表达：
  - 元素属性
  - 命中后附加状态
  - 目标优先级
  - 主动技能形态
  - 大招的控场、爆发或救场语义
- 自动战斗场景下，法系职业必须同时定义：
  - 敌方目标选择倾向
  - 友方施放对象选择倾向
  - 何时保留技能、何时立即释放
- 所有成长入口统一由 `Feat` 承载，后续落地时只使用 `grant_skill`、`modify_skill`、`replace_skill`。
- 每个法系职业的完整战斗套件统一收敛为 `4` 个技能位：
  - `普攻`
  - `核心被动`
  - `主动`
  - `大招`
- `Lv1` 固定获得 `普攻 + 核心被动`，用于建立开局职业辨识度；`主动` 与 `大招` 在后续成长节点解锁。
- 法系职业的大招不追求翻桌，而是放大该职业主循环：
  - 法师放大控场
  - 术士放大点燃爆发
  - 牧师放大救场与守护
  - 邪术师放大印记与连锁压制

## 母技能映射

- 法师：寒霜射线
- 术士：基础火焰法术
- 牧师：基础神术
- 邪术师：邪能冲击

## 四技能结构

- 所有法系职业都采用统一的 `4 技能位` 结构：
  - `普攻`：基础法术母技能，`runtime.active`
  - `核心被动`：定义职业主循环，`runtime.passive`
  - `主动`：主力手动/自动施放技能，`runtime.active`
  - `大招`：终局高潮技能，通常为 `runtime.active`
- 推荐解锁顺序：
  - `Lv1`：普攻 + 核心被动
  - `Lv3`：主动
  - `Lv5` 或终局节点：大招
- 设计目的：
  - 开局立即建立法系职业差异
  - 中期提供代表性施法按钮
  - 后期补足职业高潮与战场控制感
  - 保持自动战斗下的施法逻辑可理解、可预测

## 职业核心技

### 法师：寒霜迟滞

#### 职业核心技

- `寒霜迟滞` 的职业定位是稳定控速与冻结铺垫。
- 法师的主循环不是纯伤害，而是先拖慢目标，再用主动和大招兑现控制收益。

#### 触发规则

- 触发条件：你的寒霜法术命中目标后。
- 结算时机：本次伤害结算完成后，给目标附加减速。
- 持续时间：直到目标下回合开始前。
- 次数限制：不限制次数，但同来源重复施加时只刷新持续时间。

#### 目标选择倾向

- 普攻优先攻击高威胁前排、快要出手的敌人或高机动目标。
- 主动 `冻结新星` 优先在前排有 `2` 名及以上敌人时释放。
- 大招 `暴风雪` 优先在敌方仍有 `3` 名及以上存活单位时释放。
- 若场上已有被减速的高威胁目标，优先持续压制该目标。

#### 数值口径

- `寒霜迟滞` 建议固定为 `Speed -2`。
- `冻结新星` 建议 `CD3`，伤害低于纯输出职业同档主动，但附带稳定控制威胁。
- `暴风雪` 的强度来源应以群体减速和状态联动为主，而不是极端高伤。

#### 实现备注

- 法师优先复用寒霜射线母技能，不另造独立小冰弹体系。
- 减速建议建模为标准状态，供主动和大招检查“目标是否已减速”。
- 自动战斗 AI 需要感知敌方即将行动或高机动单位，避免冰系控制浪费在无关目标上。

### 术士：余烬点燃

#### 职业核心技

- `余烬点燃` 的职业定位是稳定挂 DOT，并围绕点燃目标兑现条件爆发。
- 术士的主循环是“先点燃，再爆燃”，而不是每个技能都做孤立直伤。

#### 触发规则

- 触发条件：你的火焰法术使目标未通过豁免后。
- 次数限制：每回合 `1` 次。
- 结算时机：本次法术伤害结算后，为目标附加燃烧状态。
- 持续时间：到目标下回合结束。

#### 目标选择倾向

- 普攻优先攻击尚未点燃的高威胁目标。
- 主动 `灰烬爆燃` 优先攻击已被点燃的目标。
- 大招 `烈焰风暴` 优先在敌方数量较多，或场上已有多个点燃目标时释放。
- 若敌方只剩单体高血量目标，则优先维持“点燃 -> 爆燃”的单点循环。

#### 数值口径

- DOT 建议起步为 `1d4`，避免点燃本身成为主要伤害来源。
- `灰烬爆燃` 的额外收益建议固定为“已点燃目标额外 `1d8`”。
- `烈焰风暴` 的核心收益应是“点燃联动的群体兑现”，而不是无条件全屏高伤。

#### 实现备注

- 点燃状态应带来源标记，避免多术士互相覆盖时语义混乱。
- `余烬点燃` 建议作为标准 DOT 状态，后续所有术士技能只检查“是否已点燃”。
- 自动战斗 AI 必须优先识别已点燃目标，否则主动技能会频繁打空联动价值。

### 牧师：神恩庇护

#### 职业核心技

- `神恩庇护` 的职业定位是持续稳场和第一时间保护队友。
- 牧师的主循环不是单纯奶量，而是“减伤托底 + 关键抬血 + 团队圣域”。

#### 触发规则

- 触发条件：每回合第一次有友军被攻击命中时。
- 结算时机：在该次伤害结算时直接减少伤害。
- 次数限制：每回合 `1` 次。
- 作用目标：默认优先作用于本回合第一个吃到攻击伤害的友军。

#### 目标选择倾向

- 普攻在团队血线稳定时，优先攻击高威胁敌人或不死/邪恶单位。
- 主动 `治愈之言` 优先施放给生命值最低或身上有关键负面状态的友军。
- 大招 `圣域祷言` 优先在己方前排承压、多人半血或敌方准备爆发时释放。
- 若团队血量稳定且无紧急救场需求，牧师应优先保留主动给下一次危险波峰。

#### 数值口径

- `神恩庇护` 建议固定为 `1d6` 减伤。
- `治愈之言` 建议为单体高质量治疗，并附带 `1` 个负面清除。
- `圣域祷言` 持续 `2` 回合即可，不建议做成长时常驻光环。

#### 实现备注

- 牧师是唯一明确需要同时设计友军目标选择与施法时机的法系职业。
- `治愈之言` 建议内置“生命最低友军”选择，不额外增加自动战斗的目标判断负担。
- `圣域祷言` 建议建模为团队短时 buff，而不是场地区域效果。

### 邪术师：静电印记

#### 职业核心技

- `静电印记` 的职业定位是短时标记与邪能连锁增幅。
- 邪术师的主循环是“先挂印记，再用雷链与雷暴收束印记收益”。

#### 触发规则

- 触发条件：你的 `邪能冲击` 命中后。
- 次数限制：每回合 `1` 次。
- 结算时机：伤害结算后给目标挂上 `静电印记`。
- 持续时间：直到你下回合结束。
- 额外收益：你本回合第一次对该目标造成伤害时，额外造成 `1d6` 雷电伤害。

#### 目标选择倾向

- 普攻优先攻击尚未带有印记的高威胁目标。
- 主动 `雷链` 优先在敌方存在相邻或密集目标时释放，并优先命中已带印记目标。
- 大招 `雷暴` 优先在敌方人数较多，或场上已经存在至少 `1` 个印记目标时释放。
- 单体残局中，优先维持“邪能冲击挂印记 -> 同目标兑现额外雷伤”的稳定循环。

#### 数值口径

- `静电印记` 的额外雷伤建议固定为 `1d6`。
- `雷链` 建议从 `1` 次弹射起步，后续成长再增加弹射数。
- `雷暴` 的群体伤害不宜过高，强度应更多来自“印记额外伤害 + 群体减速”。

#### 实现备注

- `静电印记` 必须带来源标记，避免多邪术师互相冲突。
- `雷链` 建议优先复用单体母技能结算，再做额外弹射。
- 自动战斗 AI 需要在“挂印记”和“兑现印记”之间保持闭环，否则邪术师会频繁打散输出。

## 实现落点建议

- `config/feat_build_config.lua`
  - 为四个法系职业定义 `Lv1 普攻 + 核心被动` 的固定授予
  - 后续成长统一使用 `grant_skill / modify_skill / replace_skill`
- `config/class_build_progression.lua`
  - 补齐 `Lv3 主动` 与 `Lv5 大招` 的解锁节点
- `config/skill_runtime_config.lua`
  - 为四职业的主动与大招配置 `active/passive/cd/targeting` 语义
- `config/skill_5e_meta.lua`
  - 补齐法术命中、豁免、DOT、减速、减伤、清负面等说明
- `skills/`
  - 处理 DOT、印记、团队减伤、群体法术与附加伤害逻辑

## 风险点

- 法师若把 `冻结新星` 和 `暴风雪` 都做成高伤强控，会变成压过所有法系的万能职业。
- 术士若 `点燃` 伤害本体过高，会使“点燃本身”压过主动和大招收益。
- 牧师若同时给过高奶量、清 debuff、群体减伤，容易在自动战斗中成为绝对最优后排。
- 邪术师若 `静电印记` 无来源隔离，多人同职业时会出现覆盖与重复兑现问题。
