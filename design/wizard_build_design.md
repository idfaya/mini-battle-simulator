# 法师职业设计

## 设计目标

- 本文档记录 `法师（冰）` 的职业设计，并以 `design/newbattle/caster_class_core_skill_design.md` 为规则源。
- 法师成长统一由 `Feat` 承载，不引入完整法术书、法术位恢复、长期专注槽或复杂区域状态机。
- 法师主轴是 `寒霜射线 -> 寒霜迟滞 -> 冻结新星 -> 暴风雪`，重点表达控速、冻结和行动顺序压制。
- 本文档使用三阶子职业结构：`low / mid / high`。

## 字段口径

- `promotion_stage = low / mid / high`。
- `basic_attack_slot`：寒霜射线。
- `core_slot`：寒霜迟滞。
- `mid_slot`：冻结新星。
- `high_slot`：暴风雪。
- 5e 画像：`primary_ability = int`，`spell_ability = int`，`armor_formula = robe_dex`，`save_proficiency = fort / will`（wizard 位）。

## 三阶规格

| 阶段 | 能力包 | 规则定义 |
| --- | --- | --- |
| `low` | 寒霜射线 + 寒霜迟滞 | 对单体敌人进行一次标准远程冰系法术攻击；你的寒霜法术命中后，目标直到其下回合开始前 `Speed -2`。 |
| `mid` | 冻结新星 | `CD3`，对前排敌人造成冰系伤害；目标体质豁免失败则 `STUN 1` 回合。 |
| `high` | 暴风雪 | `主动技能`，`CD5`；对全体敌人造成基础冰系伤害；命中时若目标已被减速，则伤害额外 `+1d8` 并刷新 `寒霜迟滞` 持续到其下回合开始前；命中时若目标尚未被减速，则仅施加 `寒霜迟滞`，不附加额外伤害。本技能不与 `雷暴` 共享减速来源。 |

## 核心循环

- `寒霜射线` 负责稳定单体出手。
- `寒霜迟滞` 负责给目标挂 `Speed -2`，为后续控制创造窗口。
- `冻结新星` 负责对前排进行中阶群控。
- `暴风雪` 负责在敌方已经被减速后兑现群体控场收益。

## 触发规则

- `寒霜迟滞` 在寒霜法术命中后触发。
- 同来源重复施加 `寒霜迟滞` 时只刷新持续时间，不叠加 `Speed` 惩罚。
- `Speed` 影响 `initiative` 排序与近战进入近身的行动消耗，不影响纯远程位移与法术施放距离。
- `暴风雪` 对未减速目标只挂减速，对已减速目标才追加 `+1d8` 冰系伤害。

## 目标选择倾向

- 普攻优先攻击高威胁前排、快要出手的敌人或高机动目标。
- `冻结新星` 优先在前排有 `2` 名及以上敌人时释放。
- `暴风雪` 优先在敌方仍有 `3` 名及以上存活单位，或场上有 `2` 名及以上已被减速目标时释放。
- 若场上已有被减速的高威胁目标，优先持续压制该目标。

## 数值口径

- `寒霜迟滞` 固定为 `Speed -2`。
- `冻结新星` 伤害应低于同档纯输出主动，强度来自 `STUN 1` 的威胁。
- `暴风雪` 的强度来自群体减速与已减速目标兑现，不做无条件全屏高伤。

## 实现备注

- 法师优先复用寒霜射线母技能，不另造独立小冰弹体系。
- 减速建议建模为标准状态，供 `冻结新星` 与 `暴风雪` 检查。
- 自动战斗 AI 需要感知敌方即将行动或高机动单位，避免控制浪费。

## 风险点

- `冻结新星` 与 `暴风雪` 不能同时做成高伤强控，否则法师会压过其他法系。
- 多法师场景下，`寒霜迟滞` 需要遵守同来源刷新规则，避免无限压低 `Speed`。
- 法师不做连锁弹射，避免与邪术师同质化。
