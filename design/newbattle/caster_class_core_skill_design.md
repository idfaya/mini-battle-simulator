# 法系职业核心技设计稿

## 设计目标

- 本文档统一收敛当前法系职业的核心技设计，作为后续 `Feat -> Runtime Skill -> 5e Meta` 实现的规则源。
- 目标是在保留 D&D 5e 风味的前提下，对法系职业做足够简化，使其适配当前自动战斗与轻量技能系统。
- 本文档当前覆盖仓库内已有的四个法系职业：
  - 法师（冰）
  - 术士（火）
  - 牧师（神术）
  - 邪术师（雷）
- 所有能力优先复用统一母技能，避免引入完整法术位、专注槽、法术准备列表与复杂区域状态机。
- 本文档按“低阶 / 中阶 / 高阶”三阶子职业方式拆解现有法系职业。

## 总体原则

- 5e 核心公式保持不变：
  - 攻击型法术继续使用现有法术命中链。
  - 豁免型法术继续使用现有法术豁免链。
  - 伤害、减速、点燃、减伤等效果统一挂在既有战斗结算系统上。
- 法系职业差异优先通过以下方式表达：
  - 元素属性
  - 命中后附加状态
  - 目标优先级
  - 主动技能形态
  - 高阶子职业能力的控场、爆发或救场语义
- 自动战斗场景下，法系职业必须同时定义：
  - 敌方目标选择倾向
  - 友方施放对象选择倾向
  - 何时保留技能、何时立即释放
- 所有成长入口统一由 `Feat` 承载，后续落地时只使用 `grant_skill`、`modify_skill`、`replace_skill`。
- 每个法系职业都拆成三阶子职业，与 `class_system_design.md` §2.4 的技能槽一一对应：
  - `low`：启用 `basic_attack_slot` + `core_slot`（`1 普攻 + 1 核心被动`）
  - `mid`：在 `low` 基础上追加 `mid_slot`（`1 主动` 或 `1 被动`）
  - `high`：再追加 `high_slot`（改造后的旧大招或新设计的高阶能力）
- 高阶能力不强制要求保留原大招形态；可以改写成更适合自动战斗和总纲模板的主动或被动。
- 职业成长不写成固定战斗内等级，而应通过同职业进阶、装备与 Feat 链推进。
- 法系职业的高阶能力不追求翻桌，而是放大该职业主循环：
  - 法师放大控场
  - 术士放大点燃爆发
  - 牧师放大救场与守护
  - 邪术师放大印记与连锁压制

## 字段口径

- 本文档采用 `class_system_design.md` 作为 Class 层字段规则源。
- 本文档中的职业能力对象统一为 `Class 单位`，不再使用独立“角色”口径。
- 阶段字段统一使用 `promotion_stage`。
- `promotion_stage` 的数据值统一为：
  - `low`
  - `mid`
  - `high`
- 展示层可渲染为：
  - `低阶`
  - `中阶`
  - `高阶`
- 技能槽统一使用：
  - `basic_attack_slot`
  - `core_slot`
  - `mid_slot`
  - `high_slot`
- 槽位语义统一为：
  - `basic_attack_slot` = 基础出手技能
  - `core_slot` = 低阶核心能力
  - `mid_slot` = 中阶新增能力
  - `high_slot` = 高阶终局能力
- 职业卡只推动 `Class 单位` 获得或进阶，不直接提供等级提升。
- `等级` 只负责数值成长；`进阶` 只负责同职业阶段推进；`转职` 才改变 `class_id`。

## 母技能映射

- 法师：寒霜射线
- 术士：基础火焰法术
- 牧师：基础神术
- 邪术师：邪能冲击

## 三阶子职业结构

- 所有法系职业都采用统一的 `low -> mid -> high` 三段结构：
  - `low`：建立职业辨识度，只启用 `basic_attack_slot + core_slot`
  - `mid`：在 `low` 基础上追加 `mid_slot`
  - `high`：在 `mid` 基础上追加 `high_slot`
- 推荐授予顺序：
  - 初始获得 `low`
  - 后续通过同职业进阶或关键 Feat 晋升到 `mid`、`high`
- 设计目的：
  - 开局立即建立法系职业差异
  - 中期补足职业差异和可观察联动
  - 后期补足职业高潮与战场控制感
  - 保持自动战斗下的施法逻辑可理解、可预测
  - 让高阶能力服务总纲，而不是反向改写总纲

## 一页结论

| 职业 | 核心技 | 一句话定义 |
| --- | --- | --- |
| 法师 | 寒霜迟滞 | 你的寒霜法术命中后降低目标速度，为后续控制创造窗口。 |
| 术士 | 余烬点燃 | 你的火焰法术会点燃敌人，并围绕已点燃目标兑现更高爆发。 |
| 牧师 | 神恩庇护 | 你持续为队友提供第一层防线，并在关键时刻抬血稳场。 |
| 邪术师 | 静电印记 | 你的邪能冲击会给敌人挂上短时印记，后续伤害围绕该印记放大。 |

## 四职业三阶规格

### 法师

- 5e 画像：`primary_ability = int`，`spell_ability = int`，`armor_formula = robe_dex`，`save_proficiency = fort / will`（wizard 位）。

| 阶段 | 能力包 | 规则定义 |
| --- | --- | --- |
| `low` | 寒霜射线 + 寒霜迟滞 | 对单体敌人进行一次标准远程冰系法术攻击；你的寒霜法术命中后，目标直到其下回合开始前 `Speed -2`。 |
| `mid` | 冻结新星 | `CD3`，对前排敌人造成冰系伤害；目标体质豁免失败则 `STUN 1` 回合。 |
| `high` | 暴风雪 | 保留“群体控速 + 对已减速目标额外兑现”核心的高阶控场能力。 |

### 术士

- 5e 画像：`primary_ability = int`，`spell_ability = int`，`armor_formula = robe_dex`，`save_proficiency = will`（sorcerer 位）。

| 阶段 | 能力包 | 规则定义 |
| --- | --- | --- |
| `low` | 火焰弹 + 余烬点燃 | 对单体敌人施加一次标准火焰法术豁免对抗；每回合 `1` 次，火焰法术使目标未通过豁免后，为其附加燃烧伤害。 |
| `mid` | 灰烬爆燃 | `CD3`，对当前目标施放一次强化火焰法术；若目标已被点燃，则额外造成火焰伤害。 |
| `high` | 烈焰风暴 | 保留“点燃扩散、爆燃兑现、群体连锁”核心的高阶爆发能力。 |

### 牧师

- 5e 画像：`primary_ability = str`，`spell_ability = wis`，`armor_formula = medium_capped`，`save_proficiency = will`。

| 阶段 | 能力包 | 规则定义 |
| --- | --- | --- |
| `low` | 神术裁决 + 神恩庇护 | 对单体敌人施加一次标准远程神术豁免对抗；每回合第一次有友军被攻击命中时，该次伤害减少 `1d6`。 |
| `mid` | 治愈之言 | `CD3`，为生命值最低的友军回复生命，并优先解除其 `1` 个负面状态。 |
| `high` | 圣域祷言 | 保留“团队庇护、减伤托底、关键回合稳场”核心的高阶守护能力。 |

### 邪术师

- 5e 画像：`primary_ability = int`，`spell_ability = int`，`armor_formula = robe_dex`，`save_proficiency = ref / will`（warlock 位）。

| 阶段 | 能力包 | 规则定义 |
| --- | --- | --- |
| `low` | 邪能冲击 + 静电印记 | 对单体敌人进行一次标准远程雷系法术攻击；每回合 `1` 次，邪能冲击命中后为目标附加 `静电印记`。 |
| `mid` | 雷链 | `CD3`，对当前目标造成雷电伤害，并额外弹射 `1` 名敌人；带有 `静电印记` 的目标优先成为弹射目标。 |
| `high` | 雷暴 | 保留“印记兑现、额外雷伤、群体压制”核心的高阶连锁能力。 |

## 职业核心技

### 法师：寒霜迟滞

#### 职业核心技

- `寒霜迟滞` 的职业定位是稳定控速与冻结铺垫。
- 法师的主循环不是纯伤害，而是先拖慢目标，再用中阶与高阶能力兑现控制收益。

#### 触发规则

- 触发条件：你的寒霜法术命中目标后。
- 结算时机：本次伤害结算完成后，给目标附加减速。
- 持续时间：直到目标下回合开始前。
- 次数限制：不限制次数，但同来源重复施加时只刷新持续时间。

#### 目标选择倾向

- 普攻优先攻击高威胁前排、快要出手的敌人或高机动目标。
- 中阶 `冻结新星` 优先在前排有 `2` 名及以上敌人时释放。
- 高阶 `暴风雪` 若保留主动形态，应优先在敌方仍有 `3` 名及以上存活单位时释放；若改写成被动，则应围绕已减速目标稳定兑现收益。
- 若场上已有被减速的高威胁目标，优先持续压制该目标。

#### 数值口径

- `寒霜迟滞` 建议固定为 `Speed -2`。
- `冻结新星` 建议 `CD3`，伤害低于纯输出职业同档主动，但附带稳定控制威胁。
- `暴风雪` 的强度来源应以群体减速和状态联动为主，而不是极端高伤。

#### 实现备注

- 法师优先复用寒霜射线母技能，不另造独立小冰弹体系。
- 减速建议建模为标准状态，供中阶与高阶能力检查“目标是否已减速”。
- 自动战斗 AI 需要感知敌方即将行动或高机动单位，避免冰系控制浪费在无关目标上。

### 术士：余烬点燃

#### 职业核心技

- `余烬点燃` 的职业定位是稳定挂 DOT，并围绕点燃目标兑现条件爆发。
- 术士的主循环是“先点燃，再爆燃”，而不是每个技能都做孤立直伤。

#### 触发规则

- 触发条件：你的火焰法术使目标未通过豁免后。
- 次数限制：每回合 `1` 次。
- 结算时机：本次法术伤害结算后，为目标附加燃烧状态。
- 持续时间：到目标下回合结束。

#### 目标选择倾向

- 普攻优先攻击尚未点燃的高威胁目标。
- 中阶 `灰烬爆燃` 优先攻击已被点燃的目标。
- 高阶 `烈焰风暴` 若保留主动形态，应优先在敌方数量较多或场上已有多个点燃目标时释放；若改写成被动，则 AI 只需优先维持点燃覆盖并收割点燃目标。
- 若敌方只剩单体高血量目标，则优先维持“点燃 -> 爆燃”的单点循环。

#### 数值口径

- DOT 建议起步为 `1d4`，避免点燃本身成为主要伤害来源。
- `灰烬爆燃` 的额外收益建议固定为“已点燃目标额外 `1d8`”。
- `烈焰风暴` 的核心收益应是“点燃联动的群体兑现”，而不是单纯做成无脑高伤清屏。

#### 实现备注

- 点燃状态应带来源标记，避免多术士互相覆盖时语义混乱。
- `余烬点燃` 建议作为标准 DOT 状态，后续所有术士技能只检查“是否已点燃”。
- 自动战斗 AI 必须优先识别已点燃目标，否则主动技能会频繁打空联动价值。

### 牧师：神恩庇护

#### 职业核心技

- `神恩庇护` 的职业定位是持续稳场和第一时间保护队友。
- 牧师的主循环不是单纯奶量，而是“减伤托底 + 关键抬血 + 团队圣域”。

#### 触发规则

- 触发条件：每回合第一次有友军被攻击命中时。
- 结算时机：在该次伤害结算时直接减少伤害。
- 次数限制：每回合 `1` 次。
- 作用目标：默认优先作用于本回合第一个吃到攻击伤害的友军。

#### 目标选择倾向

- 普攻在团队血线稳定时，优先攻击高威胁敌人或不死/邪恶单位。
- 中阶 `治愈之言` 优先施放给生命值最低或身上有关键负面状态的友军。
- 高阶 `圣域祷言` 可做成低频主动，也可做成跟随治疗触发的自动庇护；两种方案都应优先服务稳场，不应变成长时常驻万能光环。
- 若团队血量稳定且无紧急救场需求，牧师应优先保留主动给下一次危险波峰。

#### 数值口径

- `神恩庇护` 建议固定为 `1d6` 减伤。
- `治愈之言` 建议为单体高质量治疗，并附带 `1` 个负面清除。
- `圣域祷言` 的强度应控制在短时团队庇护，不建议做成长时常驻光环。

#### 实现备注

- 牧师是唯一明确需要同时设计友军目标选择与施法时机的法系职业。
- `治愈之言` 建议内置“生命最低友军”选择，不额外增加自动战斗的目标判断负担。
- `圣域祷言` 建议建模为团队短时 buff，而不是场地区域效果。

### 邪术师：静电印记

#### 职业核心技

- `静电印记` 的职业定位是短时标记与邪能连锁增幅。
- 邪术师的主循环是“先挂印记，再用雷链与雷暴收束印记收益”。

#### 触发规则

- 触发条件：你的 `邪能冲击` 命中后。
- 次数限制：每回合 `1` 次。
- 结算时机：伤害结算后给目标挂上 `静电印记`。
- 持续时间：直到你下回合结束。
- 额外收益：你本回合第一次对该目标造成伤害时，额外造成 `1d6` 雷电伤害。

#### 目标选择倾向

- 普攻优先攻击尚未带有印记的高威胁目标。
- 中阶 `雷链` 优先在敌方存在相邻或密集目标时释放，并优先命中已带印记目标。
- 高阶 `雷暴` 若保留主动形态，应优先在敌方人数较多或场上已经存在印记目标时释放；若改写成被动，则 AI 只需维持“挂印记 -> 兑现印记”闭环。
- 单体残局中，优先维持“邪能冲击挂印记 -> 同目标兑现额外雷伤”的稳定循环。

#### 数值口径

- `静电印记` 的额外雷伤建议固定为 `1d6`。
- `雷链` 建议从 `1` 次弹射起步，后续成长再增加弹射数。
- `雷暴` 的强度应更多来自“印记额外伤害 + 额外弹射减速”，而不是无条件全场高伤。

#### 实现备注

- `静电印记` 必须带来源标记，避免多邪术师互相冲突。
- `雷链` 建议优先复用单体母技能结算，再做额外弹射。
- 自动战斗 AI 需要在“挂印记”和“兑现印记”之间保持闭环，否则邪术师会频繁打散输出。

## 实现落点建议

- `class_system_design.md`
  - 提供 `Class 单位`、`promotion_stage`、技能槽和成长职责的统一字段口径
- `config/feat_build_config.lua`
  - 为四个法系职业定义低阶 / 中阶 / 高阶三段 Feat 授予
  - 后续成长统一使用 `grant_skill / modify_skill / replace_skill`
- `config/class_build_progression.lua`
  - 将同职业进阶、装备和 Feat 链映射到低阶 / 中阶 / 高阶的晋升
- `config/class_build_progression.lua` 中的阶段推进必须直接对应 `promotion_stage = low / mid / high`
- `skill_package_id` 的切换必须与 `basic_attack_slot / core_slot / mid_slot / high_slot` 的启用关系同步
- `config/skill_runtime_config.lua`
  - 为四职业的中阶能力与高阶能力配置 `active/passive/cd/targeting` 语义
- `config/skill_5e_meta.lua`
  - 补齐法术命中、豁免、DOT、减速、减伤、清负面等说明
- `skills/`
  - 处理 DOT、印记、团队减伤、群体法术与附加伤害逻辑

## 风险点

- 法师若把 `冻结新星` 和 `暴风雪` 都做成高伤强控，会变成压过所有法系的万能职业。
- 术士若 `点燃` 伤害本体过高，会使“点燃本身”压过中阶与高阶收益。
- 牧师若同时给过高奶量、清 debuff、群体减伤，容易在自动战斗中成为绝对最优后排。
- 邪术师若 `静电印记` 无来源隔离，多人同职业时会出现覆盖与重复兑现问题。
