# 法师职业设计

## 设计目标

- 本文档记录 `法师（冰）` 的职业设计，并以 `design/caster_class_core_skill_design.md` 为规则源。
- 法师成长统一由 `Feat` 承载，不引入完整法术书、法术位恢复、长期专注槽或复杂区域状态机。
- 法师主轴是 `寒霜射线 -> 寒霜迟滞 -> 冻结新星 -> 暴风雪`，重点表达 `霜冻 -> 冻结` 的控场链路。
- 本文档使用三阶子职业结构：`low / mid / high`。

## 字段口径

- `promotion_stage = low / mid / high`。
- `basic_attack_slot`：寒霜射线。
- `core_slot`：寒霜迟滞。
- `mid_slot`：冻结新星。
- `high_slot`：暴风雪。
- 5e 画像：`primary_ability = int`，`spell_ability = int`，`armor_formula = robe_dex`，`save_proficiency = fort / will`（wizard 位）。

## 三阶规格

| 阶段 | 能力包 | 规则定义 |
| --- | --- | --- |
| `low` | 寒霜射线 + 寒霜迟滞 | 对单体敌人进行一次标准远程冰系法术攻击；你的寒霜法术命中后，目标进入 `霜冻`：无法移动，但仍可进行远程攻击和释放技能。 |
| `mid` | 冻结新星 | `CD3`，十字范围冰系伤害；已处于 `霜冻` 的目标转为 `冻结 1` 回合，未处于 `霜冻` 的目标改为施加 `霜冻`。 |
| `high` | 暴风雪 | `主动技能`，`CD5`；对全体敌人造成基础冰系伤害；命中时若目标已 `霜冻`，则伤害额外 `+1d8` 并刷新 `霜冻`；命中时若目标尚未 `霜冻`，则仅施加 `霜冻`，不附加额外伤害。本技能不与 `雷暴` 共享状态来源。 |

## 核心循环

- `寒霜射线` 负责稳定单体出手。
- `寒霜迟滞` 负责给目标挂 `霜冻`，为后续控制创造窗口。
- `冻结新星` 负责对前排进行中阶群控。
- `暴风雪` 负责在敌方已经被 `霜冻` 后兑现群体控场收益。

## 触发规则

- `寒霜迟滞` 在寒霜法术命中后触发。
- 同来源重复施加 `寒霜迟滞` 时只刷新 `霜冻` 持续时间，不叠加其他惩罚。
- `霜冻` 的定义是“无法移动，但仍可进行远程攻击和释放技能”。
- `冻结` 的定义是 `STUN 1` 回合。
- `暴风雪` 对未 `霜冻` 目标只挂 `霜冻`，对已 `霜冻` 目标才追加 `+1d8` 冰系伤害。

## 目标选择倾向

- 普攻优先攻击高威胁前排、快要出手的敌人或高机动目标。
- `冻结新星` 优先在十字范围有 `2` 名及以上敌人，或能命中已 `霜冻` 的高威胁目标时释放。
- `暴风雪` 优先在敌方仍有 `3` 名及以上存活单位，或场上有 `2` 名及以上已 `霜冻` 目标时释放。
- 若场上已有 `霜冻` 的高威胁目标，优先持续压制该目标。

## 数值口径

- `寒霜迟滞` 固定为施加 `霜冻`，不再依赖 `Speed`。
- `冻结新星` 伤害应低于同档纯输出主动，强度来自 `STUN 1` 的威胁。
- `暴风雪` 的强度来自群体 `霜冻` 与已 `霜冻` 目标兑现，不做无条件全屏高伤。

## 实现备注

- 法师优先复用寒霜射线母技能，不另造独立小冰弹体系。
- `霜冻` 建议建模为标准状态，供 `冻结新星` 与 `暴风雪` 检查。
- 自动战斗 AI 需要感知敌方即将行动或高机动单位，避免控制浪费。

## 风险点

- `冻结新星` 与 `暴风雪` 不能同时做成高伤强控，否则法师会压过其他法系。
- 多法师场景下，`寒霜迟滞` 需要遵守同来源刷新规则，避免无限续住 `霜冻` 链。
- 法师不做连锁弹射，避免与邪术师同质化。
