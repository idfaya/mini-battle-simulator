# 5e 风格战斗系统（落地方案 v1）

## 0. 范围与定稿

本方案以 DND5e 的核心判定为目标，在 MiniBattleSimulator 现有框架（按行动推进的回合制、Timeline 技能、前后排保护、Buff/被动框架）上落地。

定稿选择（已确认）：
- 回合口径：保留现状，`turn = 每个单位行动一次就 +1`；策划与 UI 口径的“整轮”用 `turn / (aliveLeft + aliveRight)` 折算。
- 资源：保留能量条，但降级为辅助资源，不再作为主要强度来源。
- 物理：命中对抗 `AC`（护甲等级）。
- 法术：统一走 `Save vs spellDC`，不再存在 `SpellHit vs AC` 的分支。
- AOE 伤害：豁免成功默认 `半伤`。
- 强控制：豁免成功默认 `免疫控制`（不施加控制）。
- UI 名称：对外展示把 `spellDC` 叫作“法术命中”，但其含义是“法术豁免难度（DC）”。

## 1. 术语与命名

### 1.1 回合术语

- `turn`（行动回合）：某单位获得一次行动机会，系统计数 +1。
- `round`（整轮）：双方存活单位各行动一次的完整交换，用于平衡验收与 UI 展示。

### 1.2 数值命名（对外/对内）

对外显示（UI/策划文案）：
- `命中`：物理攻击命中能力（attack bonus）。
- `护甲（AC）`：决定物理是否命中。
- `法术命中`：法术豁免难度（DC，越高越难抵抗）。
- `体魄/反射/意志`：三豁免。
- `速度`：行动顺序依据（等同“先攻”）。

对内字段建议：
- `hit`：物理命中加值（等价 AttackBonus）。
- `ac`：护甲等级（Armor Class）。
- `spellDC`：法术豁免 DC（对外叫“法术命中”）。
- `saveFort/saveRef/saveWill`：三豁免加值。
- `speed`：速度。

备注：
- “法术命中”在本项目语境中不是“命中检定加值”，而是“让对方过不了豁免的难度”。UI 需要固定提示语：`法术命中：目标豁免检定需要对抗的难度（越高越难抵抗）`。

## 2. 核心判定规则（5e 骨架）

### 2.1 d20 与优势/劣势

- 普通：掷 `1d20`。
- 优势：掷 `2d20` 取高。
- 劣势：掷 `2d20` 取低。
- 同时有优势与劣势：抵消为普通 `1d20`。

天然骰规则（推荐定稿）：
- `nat 20`：必定成功（命中检定必定命中；豁免检定必定成功）。
- `nat 1`：必定失败（命中检定必定失手；豁免检定必定失败）。

### 2.2 物理攻击：命中对抗 AC

- 命中检定：`d20 + hit >= target.ac` 则命中。
- 命中后才结算伤害。

### 2.3 法术：豁免对抗法术命中（spellDC）

- 法术统一使用豁免检定：`d20 + saveX >= caster.spellDC` 则豁免成功。
- 每个法术必须指定 `saveType`：`fort/ref/will`。

法术默认结果（定稿）：
- AOE/范围伤害法术：成功半伤，失败全伤。
- 强控制法术：成功免疫控制，失败吃满控制。
- 伤害 + 强控制组合：成功半伤 + 免控，失败全伤 + 控制。

### 2.4 伤害骰与暴击

伤害统一用“骰子池 + 固定加值”表达：
- 伤害表达式：`XdY + Z`，例如 `2d6+4`。
- 多段技能：用多段表达式序列（例如 `1d8+4;1d8+4`）。

暴击（仅物理命中检定触发，默认 `nat 20`）：
- 只翻倍骰子，不翻倍固定加值：`2d6+4 -> 4d6+4`。

### 2.5 抗性/易伤/免疫

结算顺序建议：
- 先算基础伤害与暴击翻倍骰子。
- 再应用“伤害加成/减免”（若有，建议极少）。
- 最后应用抗性系统：
- `resist`：伤害减半（向下取整）。
- `vulnerable`：伤害翻倍。
- `immune`：伤害为 0。

## 3. 专注与吟唱（让命中更有威胁）

### 3.1 专注（Concentration）

适用对象：
- 强持续伤害、领域光环、强控制持续、关键增益等。

规则：
- 施法者在专注期间受到伤害时，需要做一次专注检定：
- `d20 + saveFort >= DC`。
- `DC = max(10, floor(damage / 2))`。
- 失败则掉专注，相关效果立刻结束或降级。

### 3.2 吟唱（Chant）

适用对象：
- 大范围伤害、强控制、关键团辅等（用“风险窗口”换强度）。

规则：
- 吟唱技能拥有 `chantTurns`（建议先支持 `0/1`，后续再扩展 2 段吟唱）。
- 开始吟唱时不结算技能，只进入“吟唱中”状态。
- 到施法者下次获得行动回合时，若吟唱仍存在且剩余回合为 0，则自动释放。

打断（定稿）：
- 吟唱中受到控制：直接打断。
- 吟唱中受到伤害：做一次专注检定（同 3.1），失败则打断。

被打断代价建议（默认值）：
- 消耗的能量返还 50%。
- 技能进入短冷却（例如 1 次行动回合）。

## 4. 最小属性集（落地字段）

每个单位（hero/enemy）最低需要：
- `maxHp`
- `ac`
- `speed`
- `hit`
- `spellDC`
- `saveFort/saveRef/saveWill`

可选字段：
- `critRange`（默认 20）
- `resist/vulnerable/immune`（按伤害类型）
- `concentration`（是否专注、来源技能）
- `pendingCast`（吟唱状态）

## 5. 技能配置（落地 schema）

每个技能至少声明：
- `kind`：`physical | spell`
- `resolution`：
- `physical` 固定为 `attack_roll`（对 AC）
- `spell` 固定为 `save`（对 spellDC）
- `damageType`：物理/火/冰/雷/毒/圣光/暗影等
- `damageDice`：例如 `2d6+4`（可为空表示纯控制/纯辅助）
- `saveType`：当 `kind=spell` 必填（fort/ref/will）
- `control`：可为空；若为“强控制”，标记 `controlTier=hard`
- `onSaveSuccess`：由规则自动决定：
- 若 AOE 伤害：`half`
- 若强控制：`none`
- `concentration`：true/false
- `chantTurns`：0/1/2...

建议把技能分成 4 类模板（策划填表时不易出错）：
- `physical_single`：物理单体（attack_roll）
- `physical_multi`：物理多段/连击（attack_roll 多段伤害）
- `spell_aoe_damage`：法术 AOE（save，成功半伤）
- `spell_hard_control`：法术强控（save，成功免控）

## 6. 状态系统（最小集合与规则绑定）

建议先落地这些状态，并明确它们如何影响判定：
- `stunned`：跳过行动回合；被物理攻击默认对其有优势。
- `frozen`：跳过行动回合或 Move=0；对 `saveRef` 判定劣势（可选）。
- `slowed`：speed -20% 或命中检定劣势（二选一，建议先选 speed -20%）。
- `poisoned`：物理命中检定劣势。
- `burning`：回合末触发一次伤害（建议为法术体系：`ref` 成功半伤）。
- `silenced`：不能吟唱与不能施放法术。
- `blessed`：攻击检定/豁免检定 `+1d4`。
- `cursed`：攻击检定/豁免检定 `-1d4`。
- `marked`：攻击非标记者时物理命中检定劣势（坦克价值来源）。
- `concentrating`：UI 标识。
- `chanting`：UI 标识。

## 7. 数值框架（Lv1-20 / T1-T4）

### 7.1 熟练加值（Proficiency）

- Lv1-4：+2
- Lv5-8：+3
- Lv9-12：+4
- Lv13-16：+5
- Lv17-20：+6

### 7.2 bounded accuracy 区间（终局，Lv17-20 / T4）

AC：
- 后排：13-15
- 前排：17-20
- Boss：18-21

命中（hit）：
- 后排/法系（物理命中弱）：+7-+9
- 近战输出：+9-+12
- Boss：+11-+14

法术命中（spellDC，对外叫法术命中）：
- 普通法系：16-18
- 强控制法系：18-20
- Boss：18-21

三豁免（saveFort/saveRef/saveWill）：
- 弱项：+2-+4
- 中项：+5-+7
- 强项：+8-+10

速度（speed，等同先攻）：
- 建议压缩到 90-110 的区间，避免 speed 统治一切。

### 7.3 伤害占比目标（命中就疼）

以单次命中对目标 MaxHP 的占比作为验收口径：
- 物理普攻命中：12%-18%
- 强单体命中：20%-35%
- 法术 AOE：失败 10%-18%，成功 5%-9%
- DOT 单跳：5%-8%，持续 2-3 次结算

## 8. 验收与测试（落地检查清单）

战斗读条与随机性：
- 命中检定存在 `HIT/MISS/CRIT`，且数值区间命中率在 55%-75%。
- 法术豁免存在 `SAVE_SUCCESS/SAVE_FAIL`，AOE 成功半伤，强控成功免控。
- 专注与吟唱打断有效，且在日志/UI 中可见。

节奏（以 round 折算）：
- 3v3：约 3-5 轮结束。
- 3v4：约 2.5-4.5 轮结束。

## 9. 迁移路线（不一次推翻）

- Phase 1：物理命中对 AC + 伤害骰（保留旧伤害公式作为 fallback）。
- Phase 2：法术全部迁移为 save vs spellDC。
- Phase 3：补齐优势/劣势来源（状态/站位/技能）。
- Phase 4：接入专注与吟唱，并在 UI 展示。
