# 职业 Build 优化参照

## 目的

- 本文档用于把战士最新设计与实现沉淀成通用规则，作为后续其他职业优化的统一依据。
- 目标不是复制战士技能本身，而是复用战士已经验证过的能力边界、实现路径和可观测性要求。

## 适用范围

- `Ranger`
- `Paladin`
- `Monk`
- 后续仍在旧链路上的其他职业

## 一页结论

- 所有成长统一由 `Feat` 承载。
- `skill` 是唯一能力单元。
- 运行时只保留 `active` 与 `passive`。
- 主职业玩法优先围绕一个“母技能”展开。
- `额外攻击` 和 `额外行动` 必须严格区分。
- 反应技优先采用“先登记、后结算”。
- Web 表现必须和规则语义一一对应，不能只靠日志兜底。

## 战士样板给出的硬规则

### 1. 先定母技能

- 战士的母技能是 `基础武器攻击`。
- 后续职业也必须先确定自己的母技能，再决定所有分支强化如何围绕它展开。
- 推荐映射：
  - `Ranger`：远程基础武器攻击
  - `Paladin`：近战基础武器攻击
  - `Monk`：徒手打击

### 2. 额外攻击不是额外行动

- `额外攻击` 的标准语义是：
  - 同一次基础攻击行动中的第二击
  - 默认沿用同一目标
  - 独立进行命中判定和伤害结算
  - 不重新开启新的攻击行动
- 如果一个能力允许重新选目标、重新触发整条攻击链，它就不是 `额外攻击`，而是新的完整行动。

### 3. 新行动必须显式建模

- 战士 `动作激增` 已经证明：
  - “立刻再打一刀”和“立刻获得一次新的完整攻击行动”不是一回事。
- 对其他职业来说：
  - 若设计目标是重开整条攻击链，必须在文档和实现中显式写成“额外行动”或“新的完整攻击行动”。
  - 这种能力应允许重新选择目标，并重新触发挂在母技能上的相关被动。

### 4. 反应技优先登记

- 对反击、护卫、拦截、报复类能力，优先采用：
  - 条件满足时先登记
  - 当前敌方动作结束后再结算
- 好处：
  - 覆盖未命中场景
  - 保证动画顺序稳定
  - 日志顺序更清晰

### 5. 守线 archetype 必须同时有防护与反打

- 战士 `护卫` 的最终闭环是：
  - `AC +2`
  - 熟练减伤
  - 近战条件下的登记反击
- 因此其他职业若要做“守护线”，不能只有输出反制，必须同时有明确的团队防护收益。

## 对其他职业的直接约束

### Ranger

- 若文档写“获得额外攻击”，默认解释为同一攻击行动中的第二击。
- 若还要附带 `猎人印记`、后排增伤、束缚等效果，应明确这些效果挂在第一击、第二击还是命中后的追打上。
- 不要再把“第一次攻击后再免费打一箭”与“额外攻击”混写为同一层语义。

### Paladin

- `额外攻击` 应作为近战母技能的统一频率强化。
- `神圣惩击`、命中回血、短时护队等效果应明确挂在：
  - 第一击命中后
  - 第二击命中后
  - 或整次攻击行动结束后
- 守护路线若要做团队前排保护，应优先复用战士 `护卫` 的闭环思路，而不是只做一个弱化版 aura。

### Monk

- `武艺`、`连击`、`额外攻击` 必须拆开。
- `额外攻击` 仍然是母技能的第二击。
- `武艺` 或 `连打` 这类追打应单独定义为命中后追加的小额追击，而不是冒充 `额外攻击`。
- 如果要做切后排或快速换目标，应设计为独立主动技能或新的行动，不要塞进 `额外攻击` 语义里。

## 文档编写约束

- Feat 文案必须直接写离散规则：
  - `AC +2`
  - `忽略 2 点 AC`
  - `每回合 1 次`
  - `追加 1 次攻击`
  - `命中后额外造成 1d6`
- 避免模糊说法：
  - “强化连击”
  - “提升压制力”
  - “额外展开一轮攻势”
- 每个关键能力至少写清：
  - 何时触发
  - 作用于谁
  - 命中前还是命中后
  - 是否重选目标
  - 是同一行动第二击，还是新的完整行动
  - 是否需要每回合限次

## Web 可观测性约束

- 同一行动第二击：
  - 表现为一次前冲中的二次碰撞，再回位。
- 波及副目标：
  - 表现为单次动作中的双目标命中特效。
- 命中修正：
  - 表现为短促特效或日志，不应伪装成完整位移。
- 反应技：
  - 登记日志先出现，真正出手在敌方动作后出现。

## Roguelike 约束

- 升级候选不能只看 `choiceGroup`。
- 若某职业下一等级是 `fixed` 节点，该固定 Feat 也必须进入三选一候选池。
- 否则职业会在成长节点上漏掉关键底盘能力。

## 推荐迁移顺序

1. 先确认该职业的母技能。
2. 再确认 `Lv3-Lv5` 的分支主动、被动和终盘质变。
3. 统一把能力改写成 `grant_skill / modify_skill / replace_skill`。
4. 明确哪些效果是：
   - 同一行动第二击
   - 新的完整行动
   - 命中前修正
   - 命中后追加
   - 登记后结算的反应技
5. 最后补 Roguelike、Web 日志和 Web 表现。

## 推荐检查清单

- 是否存在一个清晰的母技能
- 是否把 `额外攻击` 和 `额外行动` 写混
- 是否把守线能力误写成纯输出能力
- 是否有反应技但没有登记语义
- 是否缺少 Web 日志或特效表达
- Roguelike 是否会漏掉固定成长节点
- 文档、实现、测试三者是否一致
