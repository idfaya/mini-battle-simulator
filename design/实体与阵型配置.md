# MiniBattleSimulator 实体与阵型配置指南 (Entity & Formation)

本文档专门针对 `MiniBattleSimulator` 中的**实体装配（英雄与怪物）及阵型系统**提供详尽的配置说明。
战斗系统除了需要定义技能 (Skill) 和状态 (Buff) 的逻辑外，还需要将这些能力挂载到具体的实体上，并赋予它们基础的面板属性和战场初始站位。

---

## 一、 英雄数据配置 (`res_ally_info.json` & `res_ally_level.json`)

英雄系统采用“基础面板 + 等级成长 + 星级品质乘区”的设计。

### 1.1 基础面板与机制 (`res_ally_info.json`)
此文件定义了英雄出厂时的初始状态、定位和羁绊机制。完整的英雄对象包含近 40 个字段，以下是核心战斗强相关字段：

| 字段名 | 说明 | 示例 |
| :--- | :--- | :--- |
| **`ID`** | 英雄唯一标识 ID（即 `AllyID`）。 | `10101` |
| **`IsHero`** | 是否为玩家可用英雄（1为可用）。 | `1` |
| **`Class`** | 职业/排位约束。<br>`1`=前排(偏向生存), `2`=中排(均衡), `3`=后排(偏向输出) | `1` (坦克通常为前排) |
| **`bPos`** | **物理站位限制数组**。定义了该角色在 1~8 号阵型格中允许站立的具体编号。<br>例如前排格为 `[1,2,3]`，中后排为 `[4,5,6,7,8]`。 | `[1, 2, 3]` |
| **`HpBaseNum`** | 1级初始生命值基础固定值。 | `1200` |
| **`AtkBaseNum`** | 1级初始攻击力基础固定值。 | `150` |
| **`DefBaseNum`** | 1级初始防御力基础固定值。 | `80` |
| **`HpBaseRadio`** | 基础生命值系数（通常为 10000 即 100%）。 | `10000` |
| **`Prop`** | **核心进阶属性数组**。使用键值对配置，其中键对应 `ATTR_ID`。例如 `[18, 204]` 代表初始速度为 204。 | `[[18, 204], [21, 500]]` |
| **`Camp`** | 阵营配置（通常包含多个维度的标签），用于羁绊触发判定。 | `[1, 2, 0, 0]` |
| **`RelationTag`** | 英雄所属的羁绊标签 ID。 | `101` |
| **`SkillIDs`** | 英雄携带的主动/大招技能数组。结构通常为嵌套数组：`{"array": [{"array": [ID, Level]}]}`。 | `[100001, 100002]` |
| **`InitializeSkills`**| 进场时自动触发的初始化技能或被动光环数组。结构同上。 | `[200001]` |

**`Prop` 数组常用属性映射表 (`ATTR_ID`)**：
*   `18`: 速度基础 (SPD_BASE) - **注意：英雄的速度主要在这里配置**。
*   `21` / `22`: 暴击率 / 暴击伤害。
*   `26` / `27`: 格挡率 / 格挡强度。

### 1.2 英雄能量机制 (Energy/Mana)
战斗中，英雄释放大招（Type 3 技能）通常需要消耗 **100 能量**。
英雄生成时（`hero_data.lua`），会自动被赋予以下隐式属性：
*   **`MANA_INIT (15)`**: 初始能量值（决定开局是否能直接开大）。
*   **`MANA_MAX (16)`**: 能量槽上限。
*   **`MANA_GAIN_RATE (17)`**: 能量获取率加成（影响每次普攻或受击回能的效率）。

### 1.3 属性成长与品质乘区 (`res_ally_level.json`)
英雄的最终面板受多个维度的影响，最终计算公式为：
`最终属性 = (BaseNum基础值 + Level成长值) * 品质倍率 * 星级倍率`

*   **等级成长值**：英雄升级时，会根据其品质在此表中查找对应的成长参数（数组索引对应 1~6 级品质）。
*   **品质倍率 (`QUALITY_MULTIPLIERS`)**：普通(1.0)、优秀(1.1)、精良(1.2)、史诗(1.35)、传说(1.5)、神话(1.7)。
*   **星级倍率 (Star Level)**：采用线性增长公式 `1.0 + (star - 1) * 0.15`，即每提升一星，基础属性增加 **15%**。

### 1.4 英雄技能等级的动态拼接
英雄的技能等级通常与其自身等级挂钩。底层在实例化英雄时，会进行动态 ID 拼接：
*   **计算公式**：`技能等级 = math.min(5, math.floor(英雄等级 / 30) + 1)` (每30级提升1级，上限5级)。
*   **ID 拼接**：`最终技能ID = 技能ClassID * 100 + 技能等级`
*   **举例**：一个配置了 `100001` 技能的 60 级英雄，将携带 `10000103` (3级) 技能进入战斗。

## 二、 怪物数据配置 (`res_enemy.json` & `enemy_data.lua`)

怪物的配置与英雄不同，为了方便关卡策划批量生成各种强度的怪物，其属性主要由**职业模板系数**和**动态等级差**推算得出。

### 2.1 怪物基础配置 (`res_enemy.json`)
| 字段名 | 说明 | 示例 |
| :--- | :--- | :--- |
| **`ID`** | 怪物唯一标识 ID。 | `20101` |
| **`MonsterType`** | 怪物级别类型：`0`=普通, `1`=精英, `2`=BOSS。这通常作为一个最终的倍率乘区放大属性。 | `2` (Boss) |
| **`Class`** | 怪物职业倾向，决定了其属性成长的模板（见下文）。 | `1` |
| **`Size`** | **模型缩放比例**。影响战场上的视觉大小及碰撞体积（10000 = 100%）。 | `15000` (放大1.5倍) |
| **`DynamicLevel`** | **动态等级机制**。是否启用基于队伍的动态等级缩放。 | `1` |
| **`Prop`** | 属性抗性数组。配置诸如抗爆率(23)、闪避率(25)、格挡率(26)、破击率(28)等进阶属性，数值为万分比。 | `[{"id":25, "val":500}]` (5%闪避) |
| **`BasicCastPriority`**| AI 释放技能的优先级权重配置，决定怪物行动时的默认策略。 | `1` |
| **`SkillIDs`** | 怪物携带的常规技能（怪物的技能 ID 通常是**直接写死包含等级的完整 ID**，不会像英雄那样随等级动态拼接，可以通过 `ID % 1000` 推断技能大类）。 | `[131940201]` |
| **`SkillIDsBossWarning`** | Boss 特有的预警大招，用于在 UI 层显示高危技能释放倒计时和特写动画。 | `[131940301]` |

### 2.2 职业 (Class) 数值映射表
怪物被加载进战场时，系统会通过 `CalculateEnemyLevelGrowth` 算出它与关卡基准等级的差值，然后代入以下职业模板：

*   **Class 1: 前排型 (Frontline)**
    *   成长系数偏向：`HP * 1.2`, `ATK * 0.8`, `DEF * 1.1` (高血高防低攻)。
*   **Class 2: 中排型 (Midline)**
    *   成长系数偏向：`HP * 1.0`, `ATK * 1.0`, `DEF * 1.0` (均衡)。
*   **Class 3: 后排型 (Backline)**
    *   成长系数偏向：`HP * 0.8`, `ATK * 1.2`, `DEF * 0.9` (低血低防高攻)。

## 三、 阵型与站位系统 (`battle_formation.lua`)

战斗系统中的**攻击范围（目标选择）**强依赖于“排（Row）”的概念，这由实体的站位配置决定。

### 3.1 战区网格划分 (`wpType`)
系统将敌我双方的半场分别划分为固定编号的网格（通常为 1~8）：
*   **前排 (Front Row)**：网格编号 `1, 2, 3`。
*   **中排 (Middle Row)**：网格编号 `4, 5`。
*   **后排 (Back Row)**：网格编号 `6, 7, 8`。

### 3.2 实体落位逻辑
1.  **约束检查**：在开局布置或生成单位时，系统会读取实体的 `bPos` 数组（如 `[1,2,3]`）。
2.  **网格注册**：实体被放置到合法的网格中，`battle_formation.lua` 会将该实体注册到对应的 `wpType` 槽位上。
3.  **动态更新**：如果实体死亡，它所在的 `wpType` 将被标记为空。

### 3.3 战斗中的距离计算
`MiniBattleSimulator` 中没有绝对的坐标距离（如 5.0 米），而是**基于排的相对距离**。
当一个近战技能（配置为优先攻击前排）释放时：
1.  调用 `BattleFormation.GetEnemyRowCount`。
2.  检查敌方的前排（1,2,3 槽位）是否有存活单位。
3.  如果有，则目标锁定前排。
4.  如果前排死光，**攻击范围自动延伸至中排**（4,5 槽位）。
*(这也是为什么 `res_ally_info.json` 中一定要严格配置 `bPos` 的原因，它直接决定了角色承担伤害的顺位。)*

## 四、 完整实体装配配置示例

### 4.1 英雄坦克配置示例
以下展示了一个前排坦克英雄的完整配置，包含了基础面板、属性系数、隐藏的进阶属性（如速度和格挡）以及阵营羁绊：
```json
// res_ally_info.json 截取
{
    "ID": 10101,
    "Name": "重装盾卫",
    "IsHero": 1,
    "Class": 1,               // 1: 坦克/前排
    "bPos": [1, 2, 3],        // 只能放置在最前面的 1,2,3 号位
    
    // 基础面板与乘区系数
    "HpBaseNum": 2500,        // 极高的基础生命
    "AtkBaseNum": 80,         // 较低的基础攻击
    "DefBaseNum": 150,        // 极高的基础防御
    "HpBaseRadio": 10000,     // 生命系数 100%
    "AtkBaseRadio": 10000,
    "DefBaseRadio": 10000,
    
    // 隐藏进阶属性 (18:速度, 26:格挡率)
    "Prop": [[18, 95], [26, 1500]], // 速度 95，格挡率 15%
    
    // 阵营与羁绊
    "Camp": [1, 0, 0, 0],     // 归属 1 号阵营
    "RelationTag": 101,       // 羁绊组 101
    
    // 技能挂载
    "SkillIDs": {
        "array": [
            {"array": [100001, 1]}, // 主动技1
            {"array": [100002, 1]}  // 大招
        ]
    },
    "InitializeSkills": {
        "array": [
            {"array": [20101, 1]}   // 进场自带减伤被动光环
        ]
    }
}
```

### 4.2 敌方 Boss 配置示例
以下展示了一个关卡 Boss 的完整配置，包含了模型缩放、动态等级、AI 优先级以及专属的预警大招：
```json
// res_enemy.json 截取
{
    "ID": 50001,
    "Name": "深渊魔龙",
    "Class": 3,               // 3: 对应高攻低血的后排/刺客模板，但由 Boss Type 放大
    "MonsterType": 2,         // 2: Boss 类型（属性最终会获得巨大倍率加成）
    "Size": 15000,            // 模型放大 1.5 倍 (15000/10000)
    "DynamicLevel": 1,        // 开启根据玩家队伍动态适应等级
    
    // 怪物特有属性抗性配置
    "Prop": [
        {"id": 23, "val": 3000}, // 23: 抗爆率增加 30%
        {"id": 25, "val": 1000}  // 25: 闪避率增加 10%
    ],
    
    // AI 施法优先级
    "BasicCastPriority": 1,
    
    // 技能挂载 (怪物的技能ID通常写死，包含等级)
    "SkillIDs": [131940101, 131940201], // 普攻和常规主动技能
    "SkillIDsBossWarning": [131940301]  // Boss 预警大招（如：天核裂变），触发 UI 特写
}
```
