# AGENTS.md

This repository is a Lua/Unity (ToLua) game prototype with a small Node.js toolchain.
This document describes project conventions and operational rules for coding agents.

## Non-Negotiables

- 5e rules: HP/AC/hit bonus/spell DC must follow D&D 5e style rules. Do not "invent" new math without aligning with existing 5e helpers.
- Difficulty model: avoid stacking ad-hoc multipliers. Prefer budget-driven tuning via `budget.difficulty` and `budget.pressureFactor`.
- Directory rules:
  - Skill logic lives in `skills/`.
  - Roguelike runtime/system lives in `roguelike/`.
  - Core battle engine remains in `modules/`.
- Require style: use dot-path requires (example: `require("skills.battle_skill_status")`).

## Generated Web Mirror

- `web/public/lua/project/` is generated by `npm run export:lua`.
- Do not hand-edit generated mirror files; always edit the source Lua and re-export.
- New source directories must be added to `tools/export_web_lua.mjs` (`sourceDirs`) so export picks them up.

Default workflow after Lua changes:

1. Edit source Lua files.
2. Run `npm run export:lua` to refresh the Web mirror.

## Key Paths

- Roguelike config: `config/roguelike/`
- Core battle modules: `modules/`
- Skill modules: `skills/`
- Web mirror (generated): `web/public/lua/project/`

## Config Tables And EmmyLua

Large static config tables can trigger false positives in EmmyLua/LSP due to overly narrow inference.
Preferred mitigation:

- Add explicit EmmyLua types:
  - `---@alias` for literal unions (or inline union types if alias parsing is flaky).
  - `---@class` for entry schemas.
  - `---@type table<integer, Entry>` for top-level dictionaries.
- Keep runtime behavior unchanged; annotations are documentation/type hints only.

## Skill Metadata Synchronization

When changing skill behavior:

- Update `config/skill_5e_meta.lua` (5e meta rules).
- Update `config/res_skill.json` if the schema/fields used by runtime changed.

## Safety And Hygiene

- Avoid touching `web/dist/` and other build outputs for fixes; these are derived artifacts.
- Do not introduce destructive git commands (`reset --hard`, etc.).
- Prefer minimal, well-scoped changes; keep interfaces stable and use forwarding wrappers when splitting modules.

## Quick Commands

- Refresh web Lua mirror: `npm run export:lua`
